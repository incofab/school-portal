Introduction

EduManager is a full-stack school management platform that connects academic operations, finance, and communication in one coherent workflow. It is designed for institutions that need to manage admissions, classes, teaching, assessments, and results without losing visibility into payments, payroll, or compliance-driven reporting.

The backend is a Laravel domain layer that models school entities (sessions, classes, courses, students, staff) and orchestrates workflows like result processing, attendance, and messaging. The frontend is an Inertia + React interface built around task-focused screens for staff, students, guardians, and administrators, with printable outputs for results, receipts, and transcripts.

This documentation is a technical map of the system. Start with the backend to understand the business operations and HTTP surface, move to the frontend for UI flows and screens, then review the primary modules and entry points to locate how the application boots and where to extend it safely.

Project Summary

Backend
Laravel 12 and PHP 8.2 with Sanctum auth, Spatie permissions, Sentry monitoring, and DOMPDF exports. Orchestrates results, billing, admissions, and staff workflows.
Frontend
React 18 + TypeScript on Inertia, styled with Chakra UI and bundled via Vite. Includes reporting, publishing, and staff/student-facing flows.
Primary Modules
Institutions, class structure, admissions, curriculum, assessments, results, payments, payroll, notifications, messaging, and governance utilities.
Entry Points
HTTP entry at public/index.php and the SPA bootstrap at resources/js/app.tsx.
