    Current Feature Notes

    • Class result locking schema
      database/migrations/2026_04_13_000001_add_is_locked_to_class_result_info_table.php adds is_locked to class_result_info with a default of false. The model surface is app/Models/ClassResultInfo.php and the frontend type surface is resources/js/types/models.ts.
    • Academic session manager data
      AcademicSession remains the canonical model for school sessions and is now managed through the manager admin resource. Tests use database/factories/AcademicSessionFactory.php and cover the manager controller in tests/Feature/Http/Controllers/Managers/AcademicSessions/AcademicSessionControllerTest.php.
    • Theory question data
      TheoryQuestion belongs to its target through courseable_type and courseable_id, stores question_no, optional question_sub_number, marks, answer, and optional marking_scheme, and appears in exam payloads as theory_questions.
    • Exam courseable theory data
      ExamCourseable stores objective score/num_of_questions, theory awarded score in theory_score, obtainable marks in theory_max_score, question count in theory_num_of_questions, per-question teacher scores in theory_question_scores, and completion state in theory_evaluated. Exam mirrors theory awarded/max totals and whether every theory subject has been evaluated.
